# Tirap District Performance Index — Department KPI Data Collection Guide

**District:** Tirap, Arunachal Pradesh  
**Purpose:** Single reference document for all departments to know what data to report each month  
**Total indicators:** 124 across 14 departments  
**Version:** June 2026

Indicators are aligned with **Indian government programmes and district administration practice** — e.g. NHM/AB-PMJAY, NIPUN Bharat, ICDS/POSHAN, Swachh Bharat/ODF+, PMGSY/JJM, DISCOM/RDSS, PM-Kisan/KCC, PMAY-G, NSAP, Jan Suvidha/ILP, DARPAN grievance redressal, and DC office governance norms.

---

## 1. How to use this document

Each department head (HoD) should:

1. Find your department section below (D01 to D14).
2. Collect the figures listed for the **previous calendar month** (e.g. on 1 June, report May data).
3. Enter **numbers only** — no symbols like %, nos, or text.
4. Use the **exact unit** shown in the table.
5. Submit by the **1st of each month, 11:00 PM** (or as directed by DC Office).

**What each department fills for every indicator:**

| Field | Description |
|-------|-------------|
| Reporting Month | e.g. `May-2026` |
| Actual Value | The achieved figure in the unit specified |
| Submitted By | Name of officer submitting |
| Designation | e.g. CMO, DEO, DPO |
| Remarks | Optional — source, caveat, or explanation |

## 2. General rules

### 2.1 Units

| Unit | Meaning | How to enter |
|------|---------|--------------|
| % | Percentage | Number from 0 to 100 (e.g. `78.5`) |
| count | Whole number | Integer (e.g. `45`) |
| days | Days | Number of days (e.g. `3`) |
| hours | Hours | Number of hours (e.g. `120`) |
| rate | Per lakh population | Rate value (e.g. `150`) |
| acres | Land area | Area in acres (e.g. `25`) |
| ratio | Ratio | Ratio value (e.g. `2.5`) |
| yes/no | Binary | `1` = Yes, `0` = No |

### 2.2 Reporting frequency

| Type | What to report |
|------|----------------|
| **Monthly** | Enter the figure for that month only |
| **Quarterly** | Enter this month's progress; DC office sums months in the quarter where applicable |

### 2.3 Direction (scoring)

| Label | Meaning |
|-------|---------|
| **Higher is better** | Achieve at or above the target |
| **Lower is better** | Keep at or below the target (deaths, backlog, outages, vacancies, etc.) |

### 2.4 Cumulative indicators

Some indicators are marked **(cumulative)**. Report the **running total for the financial year** up to that month, not just the current month's addition.

## 4. Monthly submission checklist

Each department should confirm before submitting:

- [ ] Reporting month is the **previous calendar month**
- [ ] All indicators for my department are filled
- [ ] Values are **numbers only** (no % sign or text)
- [ ] Units match the table (% / count / days / hours / etc.)
- [ ] Cumulative indicators show **FY running total**
- [ ] Name and designation are recorded
- [ ] Submitted before deadline (1st of month, 11:00 PM)
